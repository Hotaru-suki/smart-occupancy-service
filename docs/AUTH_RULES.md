# 认证与注册规则

## 登录

- 默认优先显示登录界面，注册界面需要通过“注册”按钮切换进入
- 登录页只有一个公共入口，不区分管理员登录页和用户登录页
- 用户输入用户名和密码后，系统根据账号实际角色返回登录结果
- 登录成功后才显示视频、实时状态和其他面板
- 已登录用户可在工作台内自行修改密码，普通用户和管理员都支持
- 登录失败限流按“来源 + 用户名”维度统计
- 当前默认限制策略为：在 `600` 秒窗口内连续失败达到 `8` 次后，限制 `120` 秒

## 注册

- 注册页与登录页分离，通过界面切换显示
- 注册页区分两种模式：普通用户注册、管理员注册
- 普通用户注册成功后，账号角色为 `viewer`
- 管理员注册成功后，账号角色为 `admin`
- 管理员注册必须提供有效的 `ADMIN_REGISTRATION_CODE`
- 相同用户名、相同密码、相同角色的重复注册按幂等处理

## 用户名规则

- 长度必须为 `3-32`
- 仅允许字母、数字、`.`、`_`、`-`
- 首字符必须是字母或数字
- 保存时统一转换为小写

## 密码规则

- 长度必须为 `8-128`

## 权限

- `viewer`：
  - 可查看视频
  - 可查看占用状态
  - 只读
- `admin`：
  - 拥有 `viewer` 的全部权限
  - 可查看事件和历史
  - 可执行管理写操作，例如 ROI 更新、用户角色修改

## 环境变量说明

- `AUTH_USERNAME`：
  - 系统启动时自动确保存在的默认管理员用户名
- `AUTH_PASSWORD`：
  - 默认管理员明文密码，仅建议本地开发使用
- `AUTH_PASSWORD_HASH`：
  - 默认管理员密码哈希；提供后会优先于 `AUTH_PASSWORD`
- `ADMIN_REGISTRATION_CODE`：
  - 管理员注册必须提供的注册码
- `AUTH_RATE_LIMIT_WINDOW_SEC`：
  - 登录失败计数观察窗口
- `AUTH_RATE_LIMIT_MAX_ATTEMPTS`：
  - 在观察窗口内允许的最大失败次数
- `AUTH_RATE_LIMIT_LOCK_SEC`：
  - 达到阈值后的锁定时长
