## 4. 性能测试结果

### 4.1 轻量状态查询场景（status）

| 并发 | Samples | Avg(ms) | P95(ms) | 错误率 | 熔断 |
|---:|---:|---:|---:|---:|---|
| 20 | 600 | 2.19 | 3 | 0% | 否 |

| 100 | 600 | 2.08 | 3 | 0% | 否 |
| 200 | 600 | 2.04 | 3 | 0% | 否 |
| 300 | 600 | 1.86 | 3 | 0% | 否 |

#### 分析

status 场景表现稳定，在 20~300 并发范围内：

- 平均响应时间保持在约 2ms
- P95 保持在约 3ms
- 错误率始终为 0

#### 结论

`/api/status` 作为轻量查询接口，在当前测试范围内具备良好的性能稳定性和较大性能裕量。

---

### 4.2 混合轮询场景（polling）

| 并发 | Samples | Avg(ms) | P95(ms) | 错误率 | 熔断 | 触发原因 |
|---:|---:|---:|---:|---:|---|---|
| 100 | 220773 | 106.47 | 156 | 0% | 否 | ok |
| 300 | 237291 | 297.43 | 418 | 0% | 否 | ok |
| 500 | 222465 | 529.44 | 753 | 0% | 否 | ok |
| 800 | 233181 | 810.03 | 1574 | 3.1% | 是 | error_rate>=1%; p95>=1000ms |

#### 分析

1. **100 并发**：系统整体仍可稳定运行，延迟可控
2. **300 并发**：平均响应时间接近 300ms，说明开始进入明显高负载区
3. **500 并发**：平均响应时间超过 500ms，P95 接近 753ms，性能退化趋势已非常明显
4. **800 并发**：平均响应达到 810ms，P95 超过 1.5s，错误率升至 3.1%，超过阈值并触发熔断

#### 结论

混合轮询场景下，系统在 **500~800 并发区间出现性能拐点**：

- 500 并发可视为高负载但仍可运行区间
- 800 并发进入明显不可接受性能区间
