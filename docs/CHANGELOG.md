# 版本演进说明

## 当前阶段定位

当前版本已经从“基础区域占用检测后端”演进为“开发与测试并重的完整业务原型”，覆盖了业务能力、权限体系、前端工作台、自动化测试与性能优化。

## 主要演进阶段

### 阶段 1：基础检测服务

最初版本聚焦基础后端能力：

- 提供健康检查、状态查询、事件查询
- 支持 ROI 区域检测
- 支持真实模式与 Mock 模式
- 支持 Redis / MySQL 基础读写

这一阶段的重点是把“区域内是否有人”这一核心业务链路跑通。

### 阶段 2：测试工程体系补齐

在基础检测服务之上，补齐了工程质量保障能力：

- 新增 `pytest + Allure` 自动化测试
- 新增 Redis / MySQL 一致性验证
- 新增 JMeter 压测场景
- 新增 Jenkins Pipeline
- 新增 mock 模式下的稳定回归思路

这一阶段让项目从“能运行”提升到“可验证、可回归、可持续集成”。

### 阶段 3：认证与实时通信升级

在后端基础能力上继续扩展：

- 新增登录、注册、会话管理
- 引入 `HttpOnly Cookie` 认证
- 前端从轮询改为 WebSocket 实时推送
- 保留 WebRTC 视频流能力
- 新增登录限流与密码修改

这一阶段把系统从开放式演示页升级成了真正需要登录才能访问的业务工作台。

### 阶段 4：权限分离与管理员能力

围绕业务场景继续补全：

- 引入 `viewer / admin` 两级权限
- 管理员可查看事件与历史
- 管理员可更新 ROI
- 管理员可修改用户角色
- 管理员可删除用户
- 管理员可批量清理 `tester_` 测试账号

这一阶段让系统具备了基础后台管理能力。

### 阶段 5：前端工作台重构

前端不再只是接口调试页，而是按企业后台工作台思路重构：

- 登录页与注册页分离
- 支持普通用户注册与管理员注册
- 登录后进入工作台
- 工作台区分普通用户与管理员视图
- 增加账号安全区域
- 管理员工作台支持用户管理和测试账号清理

这一阶段提升了项目的演示完整度和业务表达能力。

### 阶段 6：结构与性能优化

在功能逐渐齐全后，继续优化可维护性和性能：

- 新增 `MonitoringService` 统一状态/事件缓存回退逻辑
- 新增 `HistoryService` 统一历史查询逻辑
- 路由层进一步瘦身
- WebSocket 仅为管理员计算事件负载
- `daily_stats` 落库增加节流窗口
- 错误请求方法测试改为参数化集中覆盖

这一阶段的重点是让代码更清晰、性能更稳、后续演进成本更低。

## 当前版本的项目画像

到目前为止，这个项目已经同时具备三种展示价值：

- 业务原型：能演示区域占用检测、实时状态、事件记录和后台管理
- 工程项目：具备认证、权限、缓存、数据库、实时通信和前后端联动
- 测试项目：具备自动化测试、压测、CI 和异常场景覆盖

## 后续可能的版本方向

- 增加更正式的部署文档和运维手册
- 补充更细粒度的 OpenAPI 示例
- 增加 WebSocket / WebRTC 端到端自动化验证
- 增加更深入的缓存稳定性与容灾专项方案
